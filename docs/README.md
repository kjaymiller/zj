# zj documentation

These docs are organized in the [Diátaxis](https://diataxis.fr) style — four kinds
of documentation, each serving a different need:

| Kind | Read it when you want to… | Start here |
|------|---------------------------|------------|
| **Tutorial** | learn `zj` by doing, from a clean machine to your first session | [tutorial.md](tutorial.md) |
| **How-to guides** | accomplish a specific task you already understand | [how-to/](how-to/) |
| **Reference** | look up an env var, config file, or dispatch rule | [reference.md](reference.md) |
| **Explanation** | understand how `zj` works and why it's built this way | [explanation.md](explanation.md) |

New to `zj`? Follow the [tutorial](tutorial.md). Already set up and hitting a
problem? Jump to [troubleshooting](how-to/troubleshooting.md).
